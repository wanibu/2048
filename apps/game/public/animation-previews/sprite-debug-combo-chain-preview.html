Combo 爽点链条 — 3 连合并触发

模拟完整视觉流程（并发模型）：玩家连续 3 次合并 → ComboSprite 横幅 + Girl.Surprise + Giant.Scream 三者 T=0 同时启动 → 150ms 后糖果从 Girl 手飞向 Giant 嘴 → 糖果吃完 Giant 闭嘴回 Default + 加分
为什么并发：data.json 的 C3 event sheet 里 3 个事件块并列执行（无 Wait），同一 tick 触发。视觉上 ComboSprite 瞬间 pop 最抢眼，所以感觉"它先"，其实是同时。
视觉模仿原游戏竖屏布局：顶部 Giant、中间棋盘、右下角 Girl，中央 ComboSprite 横幅。

时间轴（并发模型 — 基于 data.json 事件结构反推）： 
T = 0ms    ──▶  ① ComboSprite 横幅弹出（scale 0→1.3→1，200ms 弹出 + 400ms 保持 + 200ms 淡出 = 800ms 全周期）
T = 0ms    ──▶  ② Girl.Surprise 起播（10 帧 × 77ms = 770ms）
T = 0ms    ──▶  ③ Giant Tween Y 上移 -50px + 切 Scream 动画（3 帧 × 50ms = 150ms）→ 保持张嘴等糖
T = 150ms  ──▶  ④ 糖果开始逐颗飞行（5 颗，每颗 800ms，每 60ms 错开起飞）
T ≈ 1150ms ──▶  ⑤ 最后一颗糖果被 Giant 吃掉 → 闭嘴回 Default + 加积分
路径：右下角 Girl 手 → 抛物线向左上 → Giant 嘴。所有参数都可以在上方输入框调整。
