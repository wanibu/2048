# Giant 2048 完整游戏规则

---

## 一、棋盘
- 5×5 网格
- 每个格子可以放一个糖果或为空

## 二、糖果值
- 值为 2 的幂次：2, 4, 8, 16, 32, 64, 128, 256, 512, 1024, 2048, 4096, 8192, 16384
- 每个值对应 shape-sheet0.png 里的一帧精灵图

## 三、弹弓发射
- 弹弓在棋盘下方，可左右拖拽对准 5 列
- 按住弹弓 → 弹弓状态 1→2→3→4（拉满），糖果球跟着下移
- 按住时播放 slingshot1 音效
- 拖拽时显示列高亮光柱（snap 到整数列）
- 鼠标离开 5 列范围 → 光柱隐藏，松手不发射
- 松手发射糖果
- 只有点击弹弓区域才触发拉弓
- 键盘：左右选列，空格/上发射

## 四、落点规则
- 糖果从下方往上飞
- 碰到阻挡物 → 停在阻挡物**下方一格**
- 整列为空 → 停在**最顶行**（行1）
- 最底行有东西 → 进不去，返回列满
- 列满但最底行与发射值相同 → 不飞，直接合并（值×2）
- 落地时播放 slingshot2 音效

## 五、合并规则
- BFS 扫描上下左右相邻的同值方块为一组
- 2 个合并 → 值×2
- 3 个合并 → 值×4
- 4 个合并 → 值×8（即 2^(n-1)）
- 多组同时存在时，优先合并数量最多的组
- 合并时播放 collapse1 音效

## 六、合并位置规则
- 纵向：取组内**行号最小**的位置（向上靠拢）
- 横向：取**吸引方向的列**（打入的列，或链式合并中上一次结果的列）
- 旋转触发的合并：fallback 到组内最小列号

## 七、上靠规则（下落）
- 每次合并后，检查所有列
- 方块上面有空格 → 往上靠拢填补空隙
- 同列所有方块紧贴最顶部的方块或顶行，中间不留空隙
- 上靠后再检查合并

### 下落场景示例

**初始棋盘，打入4到列1，落在行2：**
```
行1: [  4,  8,  2, 64,  .]
行2: [  4,  4, 16,  2,  4]
行3: [  .,  8,  4,  ., 16]
行4: [  ., 32,  .,  ., 32]
行5: [  .,  .,  .,  .,  .]
```

**第1轮合并：** 3个4（行1列1、行2列1、行2列2）合并为16，放在(1,1)
```
行1: [ 16,  8,  2, 64,  .]
行2: [  .,  ., 16,  2,  4]  ← (2,1)和(2,2)被消除
行3: [  .,  8,  4,  ., 16]
行4: [  ., 32,  .,  ., 32]
行5: [  .,  .,  .,  .,  .]
```

**第1轮上靠：** 列2的(3,2)=8和(4,2)=32上面有空隙，往上靠拢
```
行1: [ 16,  8,  2, 64,  .]
行2: [  .,  8, 16,  2,  4]  ← 8从行3上移到行2
行3: [  ., 32,  4,  ., 16]  ← 32从行4上移到行3
行4: [  .,  .,  .,  ., 32]
行5: [  .,  .,  .,  .,  .]
```

**第2轮合并：** (1,2)=8和(2,2)=8相邻，合并为16，放在(1,2)
```
行1: [ 16, 16,  2, 64,  .]
行2: [  ., 32, 16,  2,  4]
行3: [  .,  .,  4,  ., 16]
行4: [  .,  .,  .,  ., 32]
行5: [  .,  .,  .,  .,  .]
```

**第2轮上靠：** 列2的32已紧贴上方，无变化。

**第3轮合并：** (1,1)=16和(1,2)=16相邻，合并为32。(1,2)是上一轮合并结果位置，吸引(1,1)过来 → 放在(1,2)
```
行1: [  ., 32,  2, 64,  .]
行2: [  ., 32, 16,  2,  4]
行3: [  .,  .,  4,  ., 16]
行4: [  .,  .,  .,  ., 32]
行5: [  .,  .,  .,  .,  .]
```

**第3轮下落：** 无变化。

**第4轮合并：** (1,2)=32和(2,2)=32相邻，合并为64，放在(1,2)（行号最小）
```
行1: [  ., 64,  2, 64,  .]
行2: [  .,  ., 16,  2,  4]
行3: [  .,  .,  4,  ., 16]
行4: [  .,  .,  .,  ., 32]
行5: [  .,  .,  .,  .,  .]
```

**第4轮下落：** 无变化。

**第5轮合并：** 无。**结束。**

## 八、循环规则
- 合并 → 下落 → 合并 → 下落 → ... 直到一轮里无合并也无下落
- 循环结束后才生成下一个糖果到弹弓上

## 九、旋转规则
- 左旋按钮 / A 键 → 逆时针 90°
- 右旋按钮 / D 键 → 顺时针 90°
- 旋转时数据层矩阵转置 + 视觉 Tween 200ms
- 旋转后触发合并+下落循环
- 旋转不影响弹弓上的糖果
- 旋转时播放 rotation 音效

## 十、下一个糖果预览
- 底部中间显示下一个糖果
- 打完当前糖果后，预览的糖果放到弹弓上，同时生成新的预览

## 十一、计分
- 合并产生的新值即为得分
- 最高分存 localStorage
- 链式合并每次都加分

## 十二、音效
- BGM：giantost2.webm，从菜单页循环播放，40%音量
- 拉弓：slingshot1
- 碰撞落地：slingshot2
- 合并：collapse1
- 旋转：rotation

## 十三、开始页面
- 蓝天绿地背景
- 巨人头像 4 帧眨眼动画（向右倾斜 8°）
- 帧1 停留 2-3 秒，帧2-3-4 每帧 80ms
- 2048 标题 + PLAY 按钮

## 十四、画面适配
- canvas 全屏（ENVELOP 模式）
- 所有元素相对比例定位
- 窗口 resize → 保存状态到 sessionStorage → 刷新 → 恢复棋盘/分数/操作记录

## 十五、服务端验证
- 每步操作记录：shoot / rotate / direct_merge
- 链式签名：sign = SHA256(prevSign + actionJSON)
- 游戏结束提交到 Cloudflare Workers
- Workers 重放签名链验证，合法则存入 D1 排行榜

- 石头用 border-sheet0.png 第一帧
- 随机生成在网格中
- 不可合并，阻挡移动
