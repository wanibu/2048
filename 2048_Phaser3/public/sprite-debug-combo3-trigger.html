一次出手后棋盘上发生 3 次（或更多）连锁合并

玩家射出一颗糖果，落地后引发 合并链：当一次出手产生 ≥3 次连续合并（第 1 次合并产出的新糖果再触发第 2 次，第 2 次又触发第 3 次），系统判定为"Combo x3"，启动完整爽点链条。

4. 并发 vs 串联 —— 哪种更准？

并发（本文档推荐）：所有视觉元素在触发瞬间同时启动。依赖的不是"一步一步等"而是每个元素自身的动画时长自然错开视觉焦点。C3 event sheet 的典型写法就是在一个父事件里并列多条 action，它们在同一 tick 内执行。

串联：上一个动画播完再起下一个。需要显式 Wait 或回调链。在 C3 event sheet 里用 System → Wait action 实现，但本次 data.json 里没有找到显式 Wait，所以并发模型更吻合代码结构。

6. 实现到 Phaser 的要点

1. 检测 3 连击：
在 checkMerges() 里维护一个 comboCount，每次链式合并 +1；合并停止时判断 combo 阈值。

2. 并发触发 3 个视觉事件：
if (comboCount >= 3) {
  // ① ComboSprite 弹出
  this.showComboBanner();
  // ② Girl.Surprise
  this.girl.play('Surprise');
  // ③ Giant Tween + Scream
  this.tweens.add({ targets: giant, y: giant.y - 50, duration: 300, yoyo: true, ease: 'Cubic' });
  giant.play('Scream');
  // ④ 生成糖果 + Bullet
  this.spawnCandiesToGiant(comboCount);
}
3. 糖果飞行：每颗用 Phaser Tween（path）从 Girl 手到 Giant 嘴，错开 60ms 起飞。

4. 全部到达后：addScore(comboBonus) + giant.play('Default')
